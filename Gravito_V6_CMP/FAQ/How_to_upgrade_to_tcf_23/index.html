Gravito CMP TCF 2.3 Upgrade

The latest Gravito CMP (v6) incorporates the TCF 2.3 changes required for disclosed vendors as well as introduces several new features, improvements, and changes to enhance user experience and compliance capabilities.

You can have a look at a sample by clicking here

New Features and Improvements

  • TCF 2.3 Compliance: Updated to comply with the latest TCF 2.3 standards, ensuring that the CMP meets current regulatory requirements including disclosed vendors section and other updates.
  • GPP (Global Privacy Platform) Support: Added support for GPP, enabling multi-region compliant CMP solutions. This enhancement allows the CMP to handle consent frameworks across multiple jurisdictions seamlessly, offering improved flexibility for global deployments.
  • Modular Architecture: The CMP has been restructured into a modular architecture, allowing for easier customization and integration of different components and frameworks based on specific needs.
  • Improved User Interface: The user interface has been revamped for better usability and accessibility, providing a more intuitive experience for end-users. This makes customization based on customer needs much more easier.
  • Enhanced Performance: Optimizations have been made to improve loading times and overall performance, ensuring a smoother experience for users.
  • Advanced Configuration Options: New configuration options have been introduced, allowing for more granular control over consent settings.
  • Tab Layout: The CMP now features a tabbed layout for easier navigation between different sections and settings. You can also add additional tabs as needed.

Note: For detailed information on the new components available in Gravito V6 CMP, please refer to the Gravito CMP V6 Components documentation.

Migration from Previous Versions

Gravito V6 introduces major architectural and UI changes compared to previous releases to make things smoother and faster. We will support you throughout the migration process to ensure a smooth, low-risk transition.

Migration Options

You have two supported migration paths — choose the one that best fits your rollout goals and resources:

  • Option A — Adopt the V6 default UI

    • Description: Use the new modern look and feel, including tabbed navigation, improved accessibility, and support for custom tabs.
    • Benefits: Full access to V6 features, easier future updates, improved UX and performance.

  • Option B — Preserve your current UI

    • Description: Keep your existing look and feel while upgrading V6 internals and compliance capabilities.
    • Benefits: Minimal UI changes for end users; reduces immediate redesign effort.

Migration steps & required information

Please follow these steps to help us prepare and execute your migration:

  1. Send us your latest in-use CMP configuration. We'll extract relevant settings and prepare a V6 configuration for you.
  2. Provide a list of all custom features, overrides, or integrations you built on top of the current CMP so we can advise on compatibility and required changes.
  3. Describe any additional changes or new features you want in the V6 CMP (UX changes, new tabs, custom workflows, textual content updates, etc.).
  4. We recommend maintaining a staging/testing environment to validate behaviors before production rollout.
  5. After we review your configurator and feature list, we'll provide a migration plan and recommended configuration changes. In majority of the cases it will be just a copy and replace of your existing CMP configuration script.

Send the required information to support@gravito.net and our team will assist with extraction, mapping, and migration guidance.

Note: If you want to create a new V6 CMP configuration from scratch instead of migrating an existing one, please visit our new CMP V6 Configurator at Gravito Admin Portal.
